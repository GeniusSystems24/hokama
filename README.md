# حكماء المستقبل

### **جدول الطلاب**
**الوصف**: يخزن معلومات حول ارتباط الطلاب بالكيانات، مما يسمح بتعيين الطلاب لكيانات محددة.

![alt text](/image/EntityStudent.svg)
- **الأعمدة**:
    - `معرف الكيان`: معرف الكيان (INT) `المفتاح الأساسي`
    - `معرف الطالب`: معرف الطالب (INT)
    - `الحالة`: حالة الارتباط (INT)، 1 لـ "نشط"، 0 لـ "غير نشط"
    - `تاريخ الإنشاء`: تاريخ إنشاء السجل (DATE)
    - `تاريخ التحديث`: تاريخ آخر تحديث للسجل (DATE)

- **العمليات**:
    - **إضافة** ارتباط جديد بين الطالب والكيان
    - **تعديل** حالة الارتباط
    - **حذف** ارتباط موجود
    - **عرض** قائمة الطلاب المرتبطين بكيان معين
    - **عرض** الكيانات المرتبطة بطالب معين

### **جدول المجموعات**
**الوصف**: يخزن معلومات حول المجموعات داخل الكيانات ويسمح بتعيين المديرين للمجموعة.

![alt text](/image/Group.svg)
- **الأعمدة**:
    - `معرف الكيان`: معرف الكيان الذي تنتمي إليه المجموعة (INT) `المفتاح الأساسي المركب & المفتاح الخارجي`
    - `معرف المجموعة`: معرف المجموعة الفريد (INT) `المفتاح الأساسي المركب`
    - `اسم المجموعة`: اسم المجموعة (VARCHAR)
    - `صورة المجموعة`: رابط لصورة المجموعة (VARCHAR)
    - `الحالة`: حالة المجموعة (INT)، مثل 1 لـ "نشط"، 0 لـ "غير نشط"
    - `تاريخ الإنشاء`: تاريخ إنشاء السجل (DATE)
    - `تاريخ التحديث`: تاريخ آخر تحديث للسجل (DATE)
    
- **العمليات**:
    - **إضافة** مجموعة جديدة
    - **تعديل** بيانات المجموعة
    - **حذف** مجموعة
    - **عرض** قائمة المجموعات بناءً على الكيان

### **جدول أعضاء المجموعة**
**الوصف**: يخزن العلاقات بين الطلاب والمجموعات التي ينتمون إليها، مما يسمح للطالب بالانتماء إلى مجموعات متعددة.

![alt text](/image/GroupMember.svg)
- **الأعمدة**:
    - `معرف الكيان`: معرف الكيان الذي تنتمي إليه المجموعة (INT) `المفتاح الخارجي المركب`
    - `معرف المجموعة`: معرف المجموعة (INT) `المفتاح الخارجي المركب`
    - `معرف العضو`: معرف العضو `المستخدم` (INT) `المفتاح الخارجي`
    - `معرف الرابط`: معرف الرابط الفريد (INT) `المفتاح الأساسي`
    - `الرتبة`: رتبة أو نوع المستخدم (VARCHAR)، مثل "مدير"، "طالب"، "مشرف"
    - `الحالة`: حالة الارتباط (INT)، مثل 1 لـ "نشط"، 0 لـ "غير نشط"
    - `تاريخ الإنشاء`: تاريخ إنشاء السجل (DATE)
    - `تاريخ التحديث`: تاريخ آخر تحديث للسجل (DATE)
    
- **العمليات**:
    - **إضافة** ارتباط جديد بين العضو والمجموعة
    - **تعديل** الارتباط
    - **حذف** ارتباط
    - **عرض** قائمة المجموعات المرتبطة بكل عضو
    - **عرض** قائمة الأعضاء المرتبطين بكل مجموعة

### **جدول الخطط**
**الوصف**: يخزن معلومات حول خطط الاشتراك المتاحة، بما في ذلك عدد المجموعات والطلاب المسموح بها، ومدة الصلاحية.

![alt text](/image/Plan.svg)
- **الأعمدة**:
    - `معرف الخطة`: معرف الخطة الفريد (INT) `المفتاح الأساسي`
    - `اسم الخطة`: اسم الخطة (VARCHAR)، مثل "الخطة الأساسية"، "الخطة المميزة"
    - `المجموعات المسموح بها`: الحد الأقصى لعدد المجموعات المسموح بها (INT)
    - `الطلاب المسموح بهم`: الحد الأقصى لعدد الطلاب المسموح بهم (INT)
    - `المدة`: مدة الصلاحية (VARCHAR)، مثل "شهر"، "سنة"
    - `السعر`: سعر الاشتراك (DECIMAL)
    - `الوصف`: وصف مختصر عن الخطة (TEXT)
    - `الحالة`: حالة الخطة (INT)، مثل 1 لـ "متاحة"، 0 لـ "غير متاحة"
    - `تاريخ الإنشاء`: تاريخ إنشاء السجل (DATE)
    - `تاريخ التحديث`: تاريخ آخر تحديث للسجل (DATE)
    
- **العمليات**:
    - **إضافة** خطة جديدة
    - **تعديل** بيانات الخطة
    - **حذف** خطة
    - **عرض** قائمة الخطط المتاحة

### **جدول الاشتراكات**
**الوصف**: يخزن معلومات الاشتراك للكيانات، بما في ذلك حزمة الميزات المتاحة مثل عدد المجموعات والطلاب.

![alt text](/image/Subscription.svg)
- **الأعمدة**:
    - `معرف الاشتراك`: معرف الاشتراك الفريد (INT) `المفتاح الأساسي`
    - `معرف الخطة`: معرف الخطة (INT) `المفتاح الخارجي`
    - `معرف الكيان`: معرف الكيان (INT) `المفتاح الخارجي`
    - `تاريخ البدء`: تاريخ بدء الاشتراك (DATE)
    - `تاريخ الانتهاء`: تاريخ انتهاء الاشتراك (DATE)
    - `حالة الاشتراك`: حالة الاشتراك (VARCHAR)، مثل "نشط"، "منتهي"
    - `الحالة`: حالة السجل (INT)، مثل 1 لـ "نشط"، 0 لـ "غير نشط"
    - `تاريخ الإنشاء`: تاريخ إنشاء السجل (DATE)
    - `تاريخ التحديث`: تاريخ آخر تحديث للسجل (DATE)
    
- **العمليات**:
    - **إضافة** اشتراك جديد
    - **تعديل** بيانات الاشتراك
    - **إلغاء** اشتراك
    - **عرض** قائمة الاشتراكات للكيانات

- **حالات الاستخدام**:
    - عند **إضافة كيان جديد**، يجب إنشاء اشتراك افتراضي له.
    - عند **تعديل كيان**، يمكن تعديل معلومات اشتراكه.
    - عند **حذف كيان**، يجب حذف الاشتراك المرتبط به.
    - التأكد من أن عدد المجموعات والطلاب لا يتجاوز الحدود المسموح بها في الاشتراك.
